## Covered

- DatePicker open state is now owned by `createDatePickerState`, including `defaultOpen`, controlled `isOpen`, and `onOpenChange`.
- Headless DatePicker tests assert default-open rendering and controlled open requests without uncontrolled mutation.
- Solid S2 DatePicker popover uses the React S2 DatePicker medium Calendar geometry for every DatePicker field size.
- Calendar popup width is asserted against React after the popover settles for S/M/L/XL DatePicker sizes.
- Calendar hover assertions now inspect the inner visible paint node while keeping the outer 32px cell geometry contract.
- Solid DatePicker popup no longer shrinks under the comparison route's S-size settled width path.
- Focused package tests and the DatePicker comparison browser suite passed.

## Remaining Gaps

- Add dedicated DatePicker route states for `maxVisibleMonths`, min/max/unavailable dates, `firstDayOfWeek`, `pageBehavior`, and form submission.
- Prove or fix date+time value composition for `granularity`, `hourCycle`, and `hideTimeZone`.
- Complete a DatePicker source branch ledger across S2, React Aria Components, React Aria datepicker hooks, and React Stately.
- Replace thresholded DatePicker field/popup visual comparisons with strict pair-diff evidence where practical.
- Add explicit assistive-technology and locale/calendar-system rows.
