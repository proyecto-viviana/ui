## Covered

- DatePicker open state is owned by `createDatePickerState`, including `defaultOpen`, controlled `isOpen`, and `onOpenChange`.
- Solid S2 DatePicker now accepts S2 string `firstDayOfWeek` values, normalizes them for headless state, and routes `maxVisibleMonths` as headless `visibleMonths`.
- Calendar min/max constraints, unavailable dates, Monday-first headers, and two visible months are asserted in both package and browser tests.
- Hidden input `name` and controlled date value are asserted through the comparison route.
- Solid S2 DatePicker popover uses the React S2 DatePicker medium Calendar geometry for every DatePicker field size.
- Calendar popup width is asserted against React after the popover settles for S/M/L/XL DatePicker sizes.
- Calendar hover assertions inspect the inner visible paint node while keeping the outer 32px cell geometry contract.
- Focused package tests and the DatePicker comparison browser suite passed.

## Remaining Gaps

- Prove or fix date+time value composition for `granularity`, `hourCycle`, and `hideTimeZone`.
- Add DatePicker route states for `form`, `validationBehavior`, and actual submit behavior.
- Complete a DatePicker source branch ledger across S2, React Aria Components, React Aria datepicker hooks, and React Stately.
- Replace thresholded DatePicker field/popup visual comparisons with strict pair-diff evidence where practical.
- Add explicit assistive-technology and locale/calendar-system rows.
