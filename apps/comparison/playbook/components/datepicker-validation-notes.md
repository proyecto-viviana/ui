## Covered

- DatePicker open state is owned by `createDatePickerState`, including `defaultOpen`, controlled `isOpen`, and `onOpenChange`.
- Solid S2 DatePicker accepts S2 string `firstDayOfWeek` values, normalizes them for headless state, and routes `maxVisibleMonths` as headless `visibleMonths`.
- Calendar min/max constraints, unavailable dates, Monday-first headers, two visible months, locale, and custom calendar creation are asserted in browser tests.
- Hidden input `name`, `form`, associated form data, React Aria default/native validation behavior, aria validation behavior, date value, date-time value, and zoned value serialization are asserted.
- `contextualHelp`, `shouldForceLeadingZeros`, and `validate` are modeled in the viewer/API notes and covered by package or browser tests.
- Solid S2 DatePicker popover uses the React S2 DatePicker medium Calendar geometry for every DatePicker field size.
- Solid S2 DatePicker popover mirrors React S2's inner wrapper and frame structure closely enough for computed surface/inner/frame/calendar token equality.
- Popup TimeField is connected to shared DatePicker state for non-day granularities.
- The deterministic closed DatePicker field route has strict normalized React-vs-Solid pair-diff evidence.
- The open calendar popover has exact-dimension React-vs-Solid pair-diff evidence with `maxMismatchRatio: 0` and `pixelThreshold: 32`; the screenshot gate normalizes each root before opening so React's z-index-auto popover is not compared while obscured by the comparison app topbar.
- Calendar popup width is asserted against React after the popover settles for S/M/L/XL DatePicker sizes.
- Calendar hover assertions inspect the inner visible paint node while keeping the outer 32px cell geometry contract.
