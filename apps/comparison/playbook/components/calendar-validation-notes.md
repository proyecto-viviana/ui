## Official Docs And Viewer Inventory

- Live official page: https://react-spectrum.adobe.com/Calendar, opened 2026-05-18.
- Primary docs example:

```tsx
import { Calendar } from "@react-spectrum/s2/Calendar";

<Calendar aria-label="Event date" />;
```

- Primary viewer controls:

| Prop             | Official surface                                                             | Route value model               | Default / omitted behavior                               |
| ---------------- | ---------------------------------------------------------------------------- | ------------------------------- | -------------------------------------------------------- |
| `visibleMonths`  | Viewer number/select control; API default `1`; S2 story options `[1, 2, 3]`. | `Default`, `1`, `2`, `3`.       | `Default` omits the prop.                                |
| `pageBehavior`   | Viewer options `single`, `visible`; API default `visible`.                   | `Default`, `single`, `visible`. | `Default` omits the prop and should behave as `visible`. |
| `firstDayOfWeek` | Viewer options `Default`, `sun`, `mon`, `tue`, `wed`, `thu`, `fri`, `sat`.   | Same order.                     | `Default` omits the prop.                                |
| `isDisabled`     | Viewer boolean control.                                                      | Switch.                         | `false`.                                                 |

- Docs examples inventoried:
  - Value: controlled `value` and `onChange`.
  - International calendars: `Provider locale="hi-IN-u-ca-indian"` and `defaultValue`.
  - Custom calendar systems: `createCalendar={() => new Custom454()}`.
  - Validation: `minValue`, `isDateUnavailable`, `isInvalid`, and `errorMessage`.
  - Controlling focused date: `focusedValue`, `onFocusChange`, and an `ActionButton`.
  - API: `visibleMonths`, `createCalendar`, `isDateUnavailable`, `isDisabled`, `isReadOnly`, `focusedValue`, `defaultFocusedValue`, `pageBehavior`, `firstDayOfWeek`, `selectionAlignment`, `styles`, `value`, `defaultValue`, and `onChange`.

## Covered

- Public root export and `./Calendar` subpath export.
- `CalendarContext` export for S2-style prop context composition.
- Official default route surface: `<Calendar aria-label="Event date" />`.
- Official viewer controls: `visibleMonths`, `pageBehavior`, `firstDayOfWeek`, and `isDisabled`.
- Controlled `value` and `onChange` selection behavior.
- Controlled `focusedValue`, `onFocusChange`, and initial `selectionAlignment` start/center/end behavior.
- Focused-date clamping for initial min/max and reactive min/max updates, with `onFocusChange` snapshots observing updated state.
- Uncontrolled single and range calendar value reset via `setValue(null)` now emits `onChange(null)`.
- Keyboard matrix parity for Arrow keys, Home/End, PageUp/PageDown, Shift+PageUp/PageDown, and ArrowRight followed by Enter selection.
- React Aria PageUp/PageDown keyboard semantics now move by one month regardless of `pageBehavior`, while next/previous buttons keep the configured paging behavior.
- Provider locale inheritance for month titles, weekday headers, localized cell aria labels, and route control serialization.
- Unicode calendar-system Provider locale route coverage for `hi-IN-u-ca-indian`, including Saka display and Gregorian emitted values.
- Custom `createCalendar` route coverage with a docs-style 4-5-4 calendar factory.
- RTL Provider direction and horizontal ArrowRight day movement under `ar-AE`.
- `firstDayOfWeek` string normalization for S2 values.
- `visibleMonths` one-, two-, and three-month route model.
- `pageBehavior` visible-vs-single paging behavior.
- `minValue`, `maxValue`, and `isDateUnavailable` route coverage.
- `isDisabled`, `isReadOnly`, `isInvalid`, and `errorMessage` route coverage.
- Calendar root semantics now match React Aria's `role="application"` plus visible-range accessible label.
- Calendar wrappers now render the screen-reader-only visible-range heading before user children.
- Invalid selected Calendar cells now expose `aria-invalid` and `aria-describedby` to the rendered error message id without adding the error id to the root description.
- Focused selectable RangeCalendar cells now expose localized React Aria-style start/finish range-selection prompt descriptions across React Aria's prompt locale set, including invalid error id composition and read-only suppression.
- Calendar browser comparison rerun for route contracts passed after the validation and root semantics assertions were added.
- Light/dark visual route smoke coverage for default and selected-day styling.
- Strict zero-tolerance React-vs-Solid pair diff now covers the deterministic default unselected grid in dark mode with `focusedValue=2025-02-15`.
- Strict zero-tolerance React-vs-Solid pair diff now covers the controlled selected-date route with `value=2025-02-03`.
- Strict zero-tolerance React-vs-Solid pair diff now covers the deterministic multi-month route with `visibleMonths=2&focusedValue=2025-02-15`.
- Default Calendar grid geometry now asserts seven columns without the trailing 48px ghost column seen in the installed React package and previous Solid port.
- Multi-month visual coverage now asserts both month grids fill the Calendar root, the heading remains compact instead of stacking, and no trailing blank calendar column appears.
- Solid Calendar title typography now uses the S2 `title-lg` style path rather than ad hoc inline text styling.
- Solid Calendar header/day foreground, selected inner paint, pressScale transform setup, and nav chevron sizing now follow the S2 token paths used by the React source.
- Solid selected calendar cells now include the selected state in their accessible names.
- Forced-colors media coverage now compares React Spectrum and Solid selected paint, default foreground, unavailable foreground, and invalid helper text foreground.
