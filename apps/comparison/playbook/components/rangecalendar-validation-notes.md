## Covered

- Styled RangeCalendar is live on both React and Solid comparison stacks.
- Interactive controls cover controlled range value, focusedValue, visible month count, page behavior, selection alignment, first day of week, min/max range constraints, unavailable dates, non-contiguous ranges, disabled, read-only, invalid, and error text.
- Solid RangeCalendar now uses S2 header/nav icons, title typography, quiet ActionButton navigation, month-grid composition, range fill/border treatment, unavailable strike, and invalid help text.
- Strict React-vs-Solid pair-diff evidence covers the constrained January grid and selected February range grid in dark mode.
- Headless RangeCalendar exposes cell wrapper styling and first/last row/column render props needed to avoid inferred calendar geometry.
- Seven-column rows and deterministic `224px` month grid geometry are asserted in package and comparison browser tests.
- Solid Stately aligns the initial visible range against min/max the same way React Stately does for the deterministic February range with a February max value.
- DateRangePicker reuses RangeCalendar without the old fixed `272px` override that could create a trailing blank gutter.

## Remaining Gaps

- Add locale/calendar-system browser rows matching the Calendar pass.
- Add explicit screen-reader/assistive-technology rows for range selection prompts.
- Complete the source-to-computed token ledger for range cell backgrounds, borders, unavailable strike, and forced-colors branches.
- Add broader route-state pair diffs beyond the deterministic constrained two-month grid.
